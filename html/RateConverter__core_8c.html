Detailed Description

Optimal-speed rate conversion cascade – implementation.

Stage selection mirrors _plan_cascade() from the validated Python prototype (src/doppler/resample/rate_converter.py, 33/33 tests).

The CIC "compound stage" bundles the CIC decimator and its optional droop-compensating FIR into a single cascade slot so that n_stages exactly equals the number of human-readable stage labels.

Intermediate ping-pong buffers (bufs[0] and bufs[1]) are grown lazily on the first execute call. buf_cap equals n_in so any decimation stage's intermediate output fits without a second alloc.

One-shot rate conversion — no persistent state required.

Creates a temporary converter, converts n_in samples, destroys it. Equivalent to:

RateConverter_state_t *rc = RateConverter_create(rate, compensate);
size_t n = RateConverter_execute(rc, in, n_in, out, max_out);
