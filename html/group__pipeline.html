Detailed Description

Push sockets distribute work across all connected Pull workers in a round-robin fashion. Unlike PUB/SUB, each frame is delivered to exactly one Pull consumer.

Receive one frame from a Pull socket (zero-copy).

On success, *msg is set to a message handle whose data buffer is valid until dp_msg_free() is called. Use dp_msg_data() to access the sample pointer.

Parameters
ctxSubscriber context.
[out]Message handleSet to a zero-copy message handle.
[out]headerSet to the frame metadata.
Returns
DP_OK on success, DP_ERR_TIMEOUT on timeout, negative on error.
